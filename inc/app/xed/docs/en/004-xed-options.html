Xed Options

The Xed widget has a few options that can be used to enhance your usage results. They are listed below:

Option Name Default Value Description
clean true Determines whether or not to use the xed.Cleaners package to clean up Xed’s output. The xed.Cleaners package uses either HTML-Tidy (preferred) or saf.HTML.Messy (default, always available) to try to ensure that the output of Xed is always XML-compliant. This is especially important if you wish to use Xed’s box and form capabilities.
tidy_path false This specifies the path to the HTML-Tidy command line utility, so that the xed.Cleaners package can use a more reliable utility than saf.HTML.Messy for its cleaning work. Note: HTML-Tidy is highly recommended over saf.HTML.Messy
images true Specifies whether or not to expose the imagechooser button to Xed users in the GUI. This allows you to disallow the inclusion of images by your users or even by certain user roles.
forms false Specifies whether or not to expose the formmaker button to Xed users in the GUI. Note: The formmaker actions are currently under development.
boxes false Specifies whether or not to expose the boxmaker button to Xed users in the GUI. Note: The boxmaker actions are currently under development.

These options can be specified as line-items in your settings.php file, or as properties of your xed.Widget.Xed $widget object.
